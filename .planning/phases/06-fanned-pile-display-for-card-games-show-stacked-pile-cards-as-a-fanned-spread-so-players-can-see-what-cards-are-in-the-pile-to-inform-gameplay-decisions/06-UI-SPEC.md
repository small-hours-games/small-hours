## Fan Layout Contract

Source: CONTEXT.md D-03, D-04, D-08, and Specifics section (solitaire-style cascade).

### Structure

- Container: `position: relative` with fixed height tall enough for 5 stacked cards.
- Each card: `position: absolute`, stacked bottom-to-top (index 0 = bottom of pile = furthest back).
- Top card (index 4, most recently played): highest `z-index`, no offset — rendered at `top: 0`.
- Each subsequent card below: offset downward by cascade step, lower `z-index`.

### Cascade Calculation

TV display:
- Container height: `(5 - 1) * 20px + 100px = 180px` (4 offsets + 1 card height)
- Card at index `i` (0 = bottom, 4 = top): `top = (4 - i) * 20px`
- `z-index`: `i + 1` (top card = 5, bottom card = 1)

Phone display:
- Container height: `(5 - 1) * 12px + 78px = 126px`
- Card at index `i`: `top = (4 - i) * 12px`
- `z-index`: `i + 1`

### Fewer Than 5 Cards

When `pileCards.length < 5`, render only the cards present. Container height is fixed (do not shrink). Cards render flush to top of container; unused slots are invisible (no placeholder cards).

### Empty Pile

Replace entire fan container with the existing empty pile placeholder:
```
display: inline-flex; align-items: center; justify-content: center;
border: 3px dashed rgba(255,255,255,0.15); border-radius: 12px;
color: var(--text-secondary); font-size: 1.2rem;
```
TV: 72px wide × 100px tall. Phone: 56px wide × 78px tall.
Copy: `Empty` (same as current — source: host.html line 1198, player.html line 1580).

### Shadow / Depth

Each fan card (except the top card) receives a subtle drop shadow on its bottom edge to separate it visually from the card below:
```
box-shadow: 0 3px 6px rgba(0, 0, 0, 0.5);
```
Top card (index 4): no shadow override — inherits `.playing-card` defaults.

Source: CONTEXT.md "Claude's Discretion" — shadow/depth effect approved as discretionary.

## Burn-Ready State

When all cards in `pileCards` share the same rank (i.e. the pile is ready to burn with a 4th card of that rank), the pile label text gains neon-purple glow:

```css
color: var(--neon-purple);
text-shadow: 0 0 8px var(--neon-purple), 0 0 20px rgba(179, 71, 217, 0.4);
```

This is a read-only visual indicator — no interaction is added to the fan itself. Players act via their hand cards as before.

## Reusability Contract

Source: CONTEXT.md D-06 — implementation should be reusable for future card games.

The fanned pile must be rendered by a dedicated function in `public/js/cards.js`:

```js
function renderFannedPile(pileCards, opts)
// pileCards: array of card objects, ordered bottom-to-top (index 0 = oldest)
// opts.cardWidth  — number (px), defaults to 72 (TV) — caller sets
// opts.cardHeight — number (px), defaults to 100 (TV)
// opts.stepPx     — vertical cascade step in px, defaults to 20 (TV)
// Returns: HTML string for the fan container (position:relative div wrapping absolute cards)
```

Host.html and player.html call this function with their respective size parameters. No duplication of layout logic between files.
