    **Step 2: Create src/fetcher/opentrivia.js** with these exact exports and internals:

    Export a single async function: `export async function fetchQuestions(categoryId, amount = 10)`

    Build the API URL: `https://opentdb.com/api.php?amount=${amount}&type=multiple`
    If categoryId is provided (truthy), append `&category=${categoryId}`

    HTML entity decoding — implement a `decodeHtml(str)` function (not exported) that handles these replacements:
    - `&amp;` -> `&`
    - `&lt;` -> `<`
    - `&gt;` -> `>`
    - `&quot;` -> `"`
    - `&#039;` -> `'`
    - `&apos;` -> `'`
    - Numeric entities like `&#123;` -> `String.fromCharCode(123)`
    Use a regex-based approach: replace `&(amp|lt|gt|quot|apos|#039|#\d+);` with a lookup map + numeric handler.

    Error code mapping — define a const object:
    ```js
    const RESPONSE_CODES = {
      0: null, // success
      1: { code: 'NO_RESULTS', message: 'Not enough questions available for this category/amount' },
      2: { code: 'INVALID_PARAMETER', message: 'Invalid category ID or amount' },
      3: { code: 'TOKEN_NOT_FOUND', message: 'Session token not found' },
      4: { code: 'TOKEN_EXHAUSTED', message: 'All questions exhausted for this token' },
      5: { code: 'RATE_LIMITED', message: 'Too many requests, please wait' },
    };
    ```

    Return shape for success: `{ ok: true, questions: [...] }`
    Return shape for API error: `{ ok: false, error: { code: string, message: string } }`
    Return shape for network error: `{ ok: false, error: { code: 'NETWORK_ERROR', message: err.message } }`

    Question transformation: map each API result to:
    ```js
    {
      id: `otdb_${index}_${Date.now()}`,
      question: decodeHtml(result.question),
      correct_answer: decodeHtml(result.correct_answer),
      incorrect_answers: result.incorrect_answers.map(a => decodeHtml(a)),
      category: decodeHtml(result.category),
      difficulty: result.difficulty,
    }
    ```

    Wrap the entire fetch + parse in try/catch. The function must NEVER throw — always return the result wrapper.
