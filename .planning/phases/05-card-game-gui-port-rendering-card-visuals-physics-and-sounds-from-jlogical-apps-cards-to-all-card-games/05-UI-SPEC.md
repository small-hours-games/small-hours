# Phase 05 — UI Design Contract

> Visual and interaction contract for the card game GUI upgrade. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — vanilla JS + HTML, no framework |
| Icon library | none — suit symbols via SVG assets |
| Font | 'Segoe UI', system-ui, -apple-system, sans-serif (from existing style.css) |

**Rationale:** The project is a no-build-step Node.js/vanilla JS server. React/shadcn do not apply. The existing design system is a dark neon theme defined as CSS custom properties in `public/css/style.css`. All new card UI must extend this existing system without introducing new tools.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between meld cards in scoring row, inline card metadata gaps |
| sm | 8px | Gap between cards in hand row (`.card-row gap`), gap between fan-layout cards |
| md | 16px | Section padding inside game panels, padding around pile display |
| lg | 24px | Vertical padding between game zones (hand zone, pile zone, scoreboard zone) |
| xl | 32px | Scoreboard layout horizontal gap between player columns |
| 2xl | 48px | Major vertical breaks between game phase sections on host display |
| 3xl | 64px | Not used in card game zones |

Exceptions:
- Card fan overlap: `-20px` left margin on phone hand (`.card-hand .playing-card` negative margin, not a spacing token)
- Card selection lift: `-12px` translateY for selected card (interaction offset, not spacing token)
- Touch target minimum: cards are 58x82px on phone (58px width exceeds 44px minimum touch target)

---

## Card Element Sizes

Two distinct sizes are used. Explicit `width`/`height` are mandatory on every `<img class="playing-card">` to prevent SVG natural-size blowout.

| Context | Width | Height | Class modifier |
|---------|-------|--------|----------------|
| Phone hand card | 58px | 82px | `.playing-card` (default) |
| Pile top / discard top (phone) | 80px | 112px | `.playing-card.pile-card` |
| Host display hand card | 48px | 64px | `.playing-card` with `width="48" height="64"` |
| Host display pile top | 68px | 96px | `.playing-card.pile-card` on host |
| Scoring meld card (small) | 40px | 56px | `.playing-card` with explicit attrs (scoring rows are dense) |

Card aspect ratio must always be 58:82 (≈ 2:3, matches SVG viewBox 261:355).

## Color

All values are pre-existing CSS custom properties from `public/css/style.css`. No new colors are introduced for this phase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a1a` (`--bg-primary`) | Page background, card game container background |
| Secondary (30%) | `rgba(20, 20, 40, 0.8)` (`--bg-card`) | Game panel glass cards, scoreboard containers |
| Accent (10%) | `#39ff14` (`--neon-green`) | Selected card border/glow, playable card hint, "your turn" indicator |
| Destructive | `#ff4757` (`--neon-red`) | Unplayable card dim state is opacity only — destructive color not used for cards |

Accent reserved for:
- Selected card outline (`box-shadow: 0 0 12px rgba(57, 255, 20, 0.4)` + `border-color: var(--neon-green)`)
- Playable card hint glow on phone controller (subtle: `box-shadow: 0 0 8px rgba(57, 255, 20, 0.15)`)
- "Your turn" active state indicator on host display

Card-specific states use the existing color vocabulary:
- Facedown card back: gradient `var(--neon-purple)` → `var(--neon-blue)` (matches existing `.playing-card.facedown` CSS)
- Win/score event: `var(--neon-blue)` (`#4dc9f6`) on scoreboard numbers
- Unplayable: `opacity: 0.35` + `filter: grayscale(0.6)` — no new color, matches existing `.playing-card.unplayable`

## Asset Contract

### SVG Card Faces

Source: `JLogical-Apps/cards` (MIT license). Downloaded once to `public/cards/faces/`.

Filename pattern: `SUIT-RANK.svg` where:
- SUIT: `CLUB`, `DIAMOND`, `HEART`, `SPADE`
- RANK: `1` through `9`, `10`, `11-JACK`, `12-QUEEN`, `13-KING`

Special mappings (enforced in `cardSvgSrc()`):
- `rank: 14` (Shithead Ace) → filename `1` (e.g. `SPADE-1.svg`)
- `rank: 1` → filename `1` (e.g. `HEART-1.svg`)
- No `ACE` suffix in filenames — Ace is always rank `1`

Facedown card: `public/cards/backs/back.svg` (plain, no variant selection in this phase)

### Sound Files

Source: `JLogical-Apps/cards` (MIT license). Downloaded once to `public/cards/sounds/`.

| File | Trigger event |
|------|--------------|
| `draw.wav` | Player draws from stock or discard pile |
| `place.wav` | Card played to pile or discarded |
| `deck_redraw.wav` | Burn pile cleared (Shithead), new hand dealt (Gin Rummy) |
| `win.wav` | Gin Rummy gin or knock win; Shithead player goes out |

Sound plays on host display (TV) only. Phone controller (`player.html`) does not play audio — TV has speakers, phones may not. `undo.wav` is not used.

## Interaction States

Every `.playing-card` element must declare all four states explicitly. No state may be left to browser defaults.

| State | Visual treatment |
|-------|-----------------|
| Default | SVG `<img>` at specified dimensions, `border-radius: 8px`, no glow |
| Hover (phone hand) | `transform: translateY(-6px)`, `z-index: 10` — lift on hover |
| Selected | `transform: translateY(-12px)`, `box-shadow: 0 0 12px rgba(57, 255, 20, 0.4)`, `outline: 2px solid var(--neon-green)` |
| Playable hint | `box-shadow: 0 0 8px rgba(57, 255, 20, 0.15)` (subtle, not selected) |
| Unplayable | `opacity: 0.35`, `filter: grayscale(0.6)`, `cursor: not-allowed`, no transform on hover/tap |
| Facedown | Show `back.svg` instead of face SVG; facedown card is not interactive unless explicitly enabled |

Transition on all state changes: `transition: transform 0.15s ease, box-shadow 0.15s ease` (matches existing `.playing-card` transition).

## Animation Contract

### CSS Flip (Gin Rummy scoring reveal)

Used when: facedown opponent cards are revealed during knock/gin scoring phase.

```
.card-flip-container { perspective: 600px; display: inline-block; }
.card-flip-inner     { transition: transform 0.4s ease; transform-style: preserve-3d; }
.card-flip-container.flipped .card-flip-inner { transform: rotateY(180deg); }
.card-face, .card-back { position: absolute; backface-visibility: hidden; -webkit-backface-visibility: hidden; }
.card-back { transform: rotateY(180deg); }
```

Duration: 400ms ease. Adding `.flipped` class to the container triggers the flip.

### Fan Layout (phone hand display)

Cards in player hand overlap left-to-right. Overlap is 20px. Selected card lifts 12px. Implemented via negative `margin-left` — no JavaScript needed.

```
.card-hand { display: flex; position: relative; padding-bottom: 12px; }
.card-hand .playing-card { margin-left: -20px; transition: transform 0.15s ease, margin 0.15s ease; }
.card-hand .playing-card:first-child { margin-left: 0; }
.card-hand .playing-card.selected { transform: translateY(-12px); }
```

### Deal Animation (host display)

Optional progressive enhancement. Cards slide in from deck position using CSS keyframe `@keyframes cardDeal`. Duration 200ms per card, staggered by 50ms per card index. If browser does not support `@keyframes` (not a concern for modern browsers), cards appear instantly — no JS fallback needed.

```
@keyframes cardDeal {
  from { transform: translateY(-20px) scale(0.9); opacity: 0; }
  to   { transform: translateY(0) scale(1); opacity: 1; }
}
.card-deal-in { animation: cardDeal 0.2s ease forwards; }
```

Stagger via inline `animation-delay: calc(var(--i) * 50ms)` where `--i` is set as `style="--i: N"` on each card element.

Physics simulation (gravity, tumbling, spring) is explicitly out of scope for this phase.

## Shared Module: `public/js/cards.js`

A single shared JS file loaded by both `host.html` and `player.html` via `<script src="/js/cards.js">`. This avoids sync drift and enables unit testing.

Exports (global functions — no module system, matches project no-build-step):
- `cardSvgSrc(c)` → string URL — maps `{rank, suit}` to SVG path
- `renderCardImg(c, opts)` → HTML string — returns full `<img>` element string

The `audio` object (Web Audio API manager) is also defined in this file and initialized on first user click.

No other shared state lives in `cards.js`. Game-specific interaction logic stays in the respective HTML files.

---

## Web Audio Contract

Single `AudioContext` instance (singleton). Initialized on first user interaction event (mandatory per browser autoplay policy). Buffers preloaded at init time.

```
audio.init()  — call inside first click/keydown listener, { once: true }
audio.play('draw')         — on draw from stock/discard
audio.play('place')        — on card played or discarded
audio.play('deck_redraw')  — on burn pile / new hand
audio.play('win')          — on game/hand win
```

Sound failure is non-fatal: `try/catch` in `_load()`, `console.warn` only. If audio fails (mobile with no user gesture, blocked AudioContext), cards render and interact normally — no error state shown to user.
